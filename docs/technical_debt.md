# 技术债与待优化清单 (Technical Debt & Improvements)

基于代码审计与项目规划，当前项目存在的已知问题、技术债及未来优化方向如下：

## 1. 交互体验 (User Experience)
| 问题描述               | 严重程度 | 改进建议                                                                                                                                                    |
| :--------------------- | :------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **缺乏图形界面 (GUI)** | High     | 目前工具完全依赖命令行运行，对非技术背景的研究人员不友好。计划引入轻量级 GUI 框架（如 PySide6 或 Streamlit）开发桌面端界面。                                |
| **配置方式原始**       | Medium   | 用户需要直接修改脚本底部的 `if __name__ == "__main__":` 代码块来调整年份、路径等参数，容易导致代码误操作。应抽离为独立的 `config.yaml` 或 `.env` 配置文件。 |

## 2. 数据存储与管理 (Data Storage)
| 问题描述           | 严重程度 | 改进建议                                                                                                                    |
| :----------------- | :------- | :-------------------------------------------------------------------------------------------------------------------------- |
| **Excel 强依赖**   | Medium   | 数据流转完全依赖 Excel 文件。随着年报数据量（尤其是全量历史数据）的增加，Excel 读写性能将显著下降，且不利于复杂查询。       |
| **缺乏数据库支持** | Medium   | 建议引入嵌入式数据库（如 SQLite 或 DuckDB）或 PostgreSQL，用于持久化存储元数据和分析结果，支持 SQL 查询与更高效的数据分析。 |

## 3. 工程化与自动化 (Engineering & DevOps)
| 问题描述         | 严重程度 | 改进建议                                                                                                           |
| :--------------- | :------- | :----------------------------------------------------------------------------------------------------------------- |
| **缺乏 CI/CD**   | Low      | 项目缺乏自动化的测试与发布流程。建议配置 GitHub Actions，实现代码提交后的自动 Lint 检查、单元测试及 Release 发布。 |
| **日志管理简陋** | Low      | 目前日志仅输出到控制台。应增加日志文件轮转（RotatingFileHandler）机制，将运行日志持久化，便于排查历史故障。        |

## 4. 扩展性与功能 (Scalability & Features)
| 问题描述         | 严重程度 | 改进建议                                                                                            |
| :--------------- | :------- | :-------------------------------------------------------------------------------------------------- |
| **云端能力缺失** | Low      | 不支持云端对象存储（如 AWS S3, OSS）或 API 形式的服务调用。未来可考虑将核心功能封装为 RESTful API。 |
| **分析维度单一** | Low      | 目前仅支持词频统计。可引入情感分析、主题模型（LDA）、文本相似度计算等更高级的 NLP 分析功能。        |
